OpenCascade学习笔记：OCC下载安装和配置

发表于2025-12-30|更新于2025-12-30|OpenCascade
|浏览量:

开发环境

  • Windows 11 （64位）
  • Visual Studio 2022
  • OpenCascade 7.9.3 (64位) （下载的版本为包含了debug的库文件和第三方的库文件的版本）

下载OCC

方式一：官网下载

  1. 访问OpenCascade开发者官网：https://dev.opencascade.org/release
  2. 导航到”Latest release”页面
  3. 选择”Windows package with debug and 3rd-party”版本并下载

方式二：GitHub下载

  1. 访问OpenCascade的GitHub网站：https://github.com/Open-Cascade-SAS/OCCT
  2. 导航到”Releases”页面
  3. 下载”opencascade-7.9.3-vc14-64-with-debug-combined.zip”

配置OCC

  1. 解压缩文件

    • 解压缩下载的压缩包
    • 压缩包包含两个文件夹：
      • opencascade文件夹：opencascade-7.9.3-vc14-64
      • 第三方库文件夹：3rdparty-vc14-64

  2. 配置OCC环境变量

    • 将opencascade-7.9.3-vc14-64\win64\vc14\bin添加到系统环境变量的PATH中

  3. 配置第三方库环境变量

    • 将3rdparty-vc14-64下需要用到的组件的bin目录添加到系统环境变量的PATH中（根据需要添加）

新建项目

  1. 创建项目

    • 创建一个新的空项目，并选择”C++”作为项目类型
    • 添加一个源文件main.cpp

  2. 配置头文件路径

    • 项目属性 → 属性配置 → C/C++ → 常规 → 附加包含目录
    • 添加OCC头文件路径：你的路径\occ根目录\inc

  3. 配置库文件路径

    • 项目属性 → 属性配置 → 链接器 → 常规 → 附加库目录
    • 添加OCC库文件路径：你的路径\occ根目录\win64\vc14\lib
    • 如果需要调试信息，使用debug版本：occ根目录\win64\vc14\libd

  4. 添加依赖库

    • 项目属性 → 属性配置 → 链接器 → 输入 → 附加依赖项
    • 添加相关的OCC库文件（按需引入）：

测试代码

运行项目

  • 编译并运行项目
  • 确保项目能够成功链接OCC库文件

可能遇到的问题

1. 提示无法打开xxxx.dll文件

解决方法：

  • 如果是第三方的dll文件：将第三方库的bin目录添加到系统环境变量的PATH中
  • 如果是occ的dll文件：将occ的bin目录添加到系统环境变量的PATH中

2. 提示无法打开xxxx.lib文件

解决方法：

  • 检查项目属性 → 属性配置 → 链接器 → 常规 → 附加库目录
  • 确保添加了正确的库文件路径
  • 如果是debug版本的库文件，需要添加debug版本的库文件路径

3. 提示无法链接到符号xxxx

解决方法：

  • 缺少对应lib文件配置
  • 需要先找出缺少的lib文件
  • 然后添加到项目属性 → 属性配置 → 链接器 → 输入 → 附加依赖项中
