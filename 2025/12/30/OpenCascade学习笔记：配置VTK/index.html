OpenCascade学习笔记：配置VTK

发表于2025-12-30|更新于2025-12-30|OpenCascade
|浏览量:

开发环境

  • Windows 11 （64位）
  • Visual Studio 2022
  • OpenCascade 7.9.3 (64位) （下载的版本为包含了debug的库文件和第三方的库文件的版本）
  • VTK 9.4.1 (64位) （occ下载的版本中包含了VTK的库文件）

配置步骤

1. 创建基础项目

  1. 新建空白的C++项目
  2. 配置OCC相关包含文件和库文件
  3. 创建一个main.cpp文件

2. 添加VTK包含目录

  • 项目属性 → 属性配置 → C/C++ → 常规 → 附加包含目录
  • 添加VTK的包含目录，例如：3rdparty-vc14-64\vtk-9.4.1-x64\include\vtk-9.4

3. 添加VTK库文件目录

  • 项目属性 → 属性配置 → 链接器 → 常规 → 附加库目录
  • 添加VTK的库文件目录，例如：3rdparty-vc14-64\vtk-9.4.1-x64\lib

4. 添加VTK库依赖项

  • 项目属性 → 属性配置 → 链接器 → 输入 → 附加依赖项
  • 添加VTK的库依赖项，例如：

5. 添加OCC对VTK支持的依赖项

  • 项目属性 → 属性配置 → 链接器 → 输入 → 附加依赖项
  • 添加OCC对VTK支持的依赖项，例如：

6. 配置VTK环境变量

方法一：项目调试环境配置

  • 配置属性 → 调试 → 环境
  • 添加：PATH=你的目录\3rdparty-vc14-64\vtk-9.4.1-x64\bin;%PATH%
  • 注意： 取消勾选”从父级或项目默认设置继承”

方法二：系统环境变量配置

  • 如果是发布版本或需要全局配置，将VTK的bin目录添加到系统环境变量PATH中

7. 添加其他第三方库路径

  • 可以添加到调试属性 → 环境中
  • 也可以添加到系统环境变量PATH中

测试代码

可能遇到的问题

1. 提示无法打开xxx.dll文件

解决方法：

  • 将无法打开的第三方的文件对应的bin目录添加到环境变量PATH中
  • 或者添加到属性 → 配置属性 → 调试 → 环境的PATH中

2. 提示无法打开xxx.lib文件

解决方法：

  • 将无法打开的lib文件对应的lib目录添加到属性 → 配置属性 → 链接器 → 常规 → 附加库目录中

3. 无法链接到符号xxx

解决方法：

  • 检查是否添加了正确的库依赖项，例如：vtkRenderingOpenGL2-9.4.lib
